Welcome to the XCaseServiceClient Application

This tool provides a means to examine and invoke the methods exposed by a Web service. The service descriptor can be either WSDL for SOAP Web services or a Swagger documnent for a REST Web service. It loads the service descriptor from either a URL or file location, and then introspects on the service descriptor to generate a UI to invoke the Web service methods.

For the Swagger selection, you can choose one of two options:

  • generate and run the UI using C# code
  • generate Java code that represents the service proxy and its API methods

You can save the details of a Web service (WSDL URL, and authentication information) as an XSVC (XCase Service) file.

Enter the URL to access the Web service WSDL. Enter authentication information if required. Click URL.

If you change any parameter, then you must click URL again to regenerate the Web service client.

File Menu

Click Open to open a previously-saved XSVC file.

Click Save to save the current settings to disk.

Click Save As to save the current settings to disk as an XSVC file.

Click Load to load the current settings from disk.

Settings Menu

Click Proxy to set or modify proxy server settings. (Not yet implemented)

Click Interface to update the UI settings for this session.
